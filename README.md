# Рекламный сайт Space Text World

Одностраничный лендинг для привлечения игроков в Telegram MMO про космос.

## Что сделать перед публикацией

1. **Ссылка на бота**  
   В `index.html` замените `https://t.me/YourBotUsername` на реальную ссылку на вашего бота. Ссылка задаётся в скрипте внизу страницы — она подставляется в обе кнопки «Играть в Telegram» (в шапке и в конце страницы).

2. **Размещение**  
   Загрузите папку на любой хостинг (GitHub Pages, Netlify, Vercel, свой сервер) или откройте `index.html` локально для проверки. Для публичной ссылки желателен HTTPS (см. ниже).

## Содержимое страницы

- **Hero:** космический фон (градиенты, звёзды), название игры, слоган, краткое описание и первая кнопка CTA.
- **Как начать:** три шага (нажать кнопку → открыть бота → играть).
- **Что внутри игры:** четыре карточки (Планета, Космос, Альянсы, Всё в Telegram).
- **Финальный CTA:** блок «Готов начать?» и вторая кнопка перехода в Telegram.

Файлы: только `index.html` (стили и скрипт внутри страницы).

## Как поделиться с друзьями

**Вариант 1: Прямая ссылка на бота**  
Ссылка у вас уже есть — та, что вы подставили в `index.html` (например `https://t.me/ВашБот`). Её можно:
- отправить в личку в Telegram, в любой чат или группу;
- вставить в статус/историю (Telegram, соцсети);
- отправить в мессенджере или по почте.

Друг нажимает на ссылку → открывается ваш бот → нажимает **Start** или пишет `/start` — и попадает в игру. Никакой сайт открывать не обязательно.

**Вариант 2: Ссылка на лендинг (сайт)**  
Если вы уже выложили сайт на GitHub Pages, Netlify или Vercel (см. раздел «Как сделать сайт с HTTPS»), у вас есть адрес вида:
- `https://ваш-логин.github.io/имя-репозитория/`
- или `https://ваш-проект.netlify.app`
- или `https://ваш-проект.vercel.app`

Этой ссылкой можно поделиться так же — в чатах, соцсетях, по почте. Друг откроет страницу с описанием игры и кнопкой «Играть в Telegram»; по нажатию откроется бот. Удобно, когда хотите сначала показать, что за игра, а уже потом вести в Telegram.

**Что отправить:** для быстрого старта друзьям достаточно **прямой ссылки на бота** (вариант 1). Ссылку на сайт имеет смысл давать, когда хочется «презентовать» игру текстом и кнопкой.

## Как сделать сайт с HTTPS

HTTPS на этих сервисах включается автоматически — отдельно настраивать сертификат не нужно.

### Вариант 1: GitHub Pages (бесплатно)

1. Создайте репозиторий на [GitHub](https://github.com) (например `stw-promo` или `space-text-world`).
2. Загрузите в репозиторий файл `index.html` — в корень репозитория или в папку `docs`, либо только папку `promo_site` и положите `index.html` в корень (важно: главная страница должна называться `index.html` и быть в корне того, что отдаётся сайтом).
3. В репозитории: **Settings** → **Pages** (в левом меню).
4. В блоке **Source** выберите **Deploy from a branch**.
5. В **Branch** выберите `main` (или `master`), папку — **/ (root)** — и нажмите **Save**.
6. Подождите 1–2 минуты. Сайт будет доступен по адресу:  
   `https://ваш-логин.github.io/имя-репозитория/`  
   Если репозиторий называется `stw-promo`, ссылка будет вида:  
   `https://username.github.io/stw-promo/`
7. Для отображения из корня (без `/stw-promo/` в пути) можно создать репозиторий с именем `username.github.io` — тогда сайт откроется по `https://username.github.io`.

**Важно:** в корне ветки должен лежать `index.html`. Если весь проект в одной папке, в корень репозитория скопируйте только `index.html` (и при необходимости подпапки со стилями/картинками, если позже добавите).

---

### Вариант 2: Netlify (бесплатно, без Git)

1. Зайдите на [netlify.com](https://www.netlify.com) и зарегистрируйтесь (можно через GitHub или email).
2. В панели нажмите **Add new site** → **Deploy manually** (или **Drag and drop**).
3. Перетащите в окно браузера папку с вашим сайтом — ту, где лежит `index.html` (например папку `promo_site` целиком, если в ней только `index.html`). Либо создайте папку, положите в неё только `index.html`, и перетащите эту папку.
4. Netlify загрузит файлы и выдаст ссылку вида `https://случайное-имя.netlify.app`. HTTPS уже включён.
5. Имя можно изменить: **Site settings** → **Domain management** → **Options** у домена → **Edit site name** — введите, например, `space-text-world.netlify.app`.

---

### Вариант 3: Netlify или Vercel через Git

1. Залейте проект в GitHub (как в варианте 1).
2. На [netlify.com](https://www.netlify.com) или [vercel.com](https://vercel.com): **Add new site** → **Import from Git** → выберите репозиторий.
3. Укажите, что публикуется статический сайт: **Build command** оставьте пустым, **Publish directory** — `promo_site` (если `index.html` лежит в папке `promo_site`) или `/` (если `index.html` в корне репозитория).
4. После деплоя сайт будет доступен по HTTPS с адресом вида `ваш-проект.netlify.app` или `ваш-проект.vercel.app`.

---

### Вариант 4: Свой домен с HTTPS

Если купили домен (например `spaceworld.game`):

- **Netlify:** **Site settings** → **Domain management** → **Add custom domain** → введите домен. Netlify выдаст инструкцию: у регистратора в DNS нужно добавить запись (A или CNAME), как указано в панели. SSL (HTTPS) Netlify включит сам (Let's Encrypt).
- **GitHub Pages:** В настройках репозитория → **Pages** → **Custom domain** — укажите домен. В настройках DNS у регистратора добавьте запись, как просит GitHub (обычно CNAME на `username.github.io`). Галочку **Enforce HTTPS** включите после того, как домен начнёт открываться.
- **Vercel:** **Settings** → **Domains** → добавьте домен и выполните шаги по настройке DNS; HTTPS выдаётся автоматически.

После сохранения настроек DNS может потребоваться от нескольких минут до 24–48 часов, пока домен начнёт открываться по HTTPS.

---

### Проверка HTTPS

Откройте ссылку на сайт в браузере. В адресной строке слева от адреса должна быть иконка замка; адрес должен начинаться с `https://`. Если видите «Подключение не защищено» или «Not secure» — значит открыт `http://` или хостинг ещё не выдал сертификат (подождите 2–5 минут после первого деплоя).

## Стоит ли в дальнейшем создать полноценный лендинг с HTTPS?

**Кратко: да, если планируете серьёзно продвигать игру и давать ссылку на сайт незнакомым людям.**

### Зачем нужен HTTPS

- **Доверие:** браузеры помечают сайты без HTTPS как «небезопасные» — часть пользователей не станет переходить по ссылке.
- **Реклама и соцсети:** многие площадки (реклама, посты) охотнее ведут на HTTPS; часть сервисов вообще не принимает http-ссылки.
- **Telegram Mini App:** если позже захотите встроить в бота Web App (например, карту космоса), Telegram требует HTTPS для открытия внешних страниц.
- **Аналитика и реклама:** корректная работа счётчиков и UTM-меток на HTTPS предсказуемее.

### Когда достаточно текущего варианта

- Раздаёте ссылку только друзьям/тестерам или в закрытых чатах.
- Используете сайт как «визитку» в описании бота или в одном-двух постах без платной рекламы.
- Пока не планируете Web App и не собираете статистику по переходам.

### Что считать «полноценным» лендингом

Не обязательно переделывать дизайн. «Полноценный» здесь — это в первую очередь:

1. **Размещение с HTTPS** — бесплатно на GitHub Pages, Netlify, Vercel (домен вида `username.github.io`, `project.netlify.app` или свой домен с бесплатным SSL).
2. **Свой домен (по желанию)** — например `spaceworld.game` вместо `username.github.io/...` — выглядит серьёзнее и проще запомнить.
3. **Один и тот же `index.html`** — текущую страницу можно просто залить на выбранный хостинг; доработки контента (тексты, скриншоты, отзывы) можно добавлять постепенно.

### Рекомендация

- **Сейчас:** можно оставить один файл `index.html` и при необходимости выложить его на GitHub Pages / Netlify / Vercel — получите HTTPS без настройки сервера.
- **Когда начнёте продвижение (реклама, посты, незнакомые люди):** обязательно перейти на хостинг с HTTPS (тот же GitHub Pages или другой) и при желании подключить свой домен. Отдельно «переделывать» сайт под HTTPS не нужно — достаточно перенести текущую страницу на хостинг с SSL.

Итого: полноценный лендинг с HTTPS имеет смысл, когда вы готовы делиться ссылкой широко или планируете Web App/аналитику; технически это чаще всего та же страница, но размещённая на сервисе с бесплатным HTTPS.
